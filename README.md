# 麦克纳姆轮语音识别智能小车
🚖 基于STM32F407VET6标准库开发的麦克纳姆轮全向智能小车，集成离线语音识别、BLE低功耗蓝牙透传、超声波自动避障、DHT11温湿度采集、四路PWM电机驱动、状态指示灯全套驱动，模块化分层代码，外设完全解耦。

## 项目简介
本项目主控为STM32F407VET6，搭载四轮麦克纳姆轮底盘，支持全向移动、原地旋转；外设集成 SU-03T 离线语音模块、HC-SR04 超声波测距、DHT11 温湿度传感器、Ai-WB2-32S BLE蓝牙透传模块、调试串口。
全部驱动独立封装 `.c/.h`，无耦合冗余代码，自带中断处理、超时防卡死、硬件自检逻辑，可直接编译下载运行，支持语音控制、安卓蓝牙APP无线控制、自动避障、环境温湿度采集。
配套安卓BLE控制APP开源仓库：https://github.com/ocfh/ble-car-controller-android-java ，可搭配本工程实现手机无线操控小车。

## 核心功能特性
### 1. SU-03T 离线语音指令控制
UART4 中断实时接收语音模块指令，无缓存延迟，直接控制麦克纳姆轮小车与灯光。

![SU-03T语音固件截图](screenshot/4.jpg)

- `0x0A`：小车前进
- `0xA0`：小车后退
- `0x43`：原地左转
- `0x44`：原地右转
- `U`：减速（默认速度300，每次-50）
- `O`：点亮LED指示灯
- `I`：熄灭LED指示灯
- `S`：LED快速闪烁3次

### 2. HC-SR04 超声波自动避障
- 避障阈值：25cm，前方障碍物距离小于阈值自动触发避障流程
- 避障动作：停车2s → 后退500ms → 循环切换前进/左右绕行，适配麦克纳姆轮全向移动特性
- 主循环100ms非阻塞测距，内置超时判断，防止程序死循环卡死
- 全局变量 `auto_avoid_flag` 一键开启/关闭自动避障功能

![小车自动避障实拍](screenshot/3.jpg)

### 3. DHT11 环境温湿度采集
- 标准单总线时序，40bit数据校验和校验，提升读取稳定性
- 支持0.1精度小数温湿度换算
- 对外提供 `Get_temperature()` / `Get_humidity()` 读取接口
- 预留串口周期打印代码，取消注释即可实时上报温湿度至串口/蓝牙

### 4. 麦克纳姆轮全向运动驱动（核心）
TIM3 四路独立PWM调速，PWM分辨率 0~1000，速度正负区分电机正反转；
底层独立控制四个轮子差速，封装基础运动函数，可自行拓展横移、斜走功能：
```c
// 基础运动API
void Car_Forward(int16_t speed);    // 直行前进
void Car_Backward(int16_t speed);   // 直行后退
void Car_Left(int16_t speed);       // 原地左转
void Car_Right(int16_t speed);      // 原地右转
void Motor_StopAll(void);           // 四轮全部停止
```

![小车正常行驶实拍](screenshot/2.jpg)

### 5. 三路串口通信
1. **USART1**：调试串口，打印温湿度、运行日志
2. **USART3**：BLE蓝牙串口，适配Ai-WB2-32S，与安卓APP双向数据交互，下发运动指令、上传温湿度/测距数据
3. **UART4**：SU-03T语音专用串口，固定9600波特率，中断接收语音指令
通用封装 `USARTx_Send_String()`，一行代码发送字符串。

### 6. 配套安卓BLE控制APP
独立开源安卓控制软件，仓库地址：https://github.com/ocfh/ble-car-controller-android-java
- BLE GATT协议通信，适配Android 12+动态蓝牙权限
- 虚拟摇杆控制小车运动、滑动条调节车速
- 实时接收下位机推送温度、超声波距离数据
- 快捷按键控制车灯、语音、测距功能

### 7. 辅助配套功能
- 开机硬件自检：LED闪烁3次，确认所有外设初始化完成

![点灯成功实拍](screenshot/1.jpg)

- 模块化初始化，不用的外设可注释关闭，节省系统资源
- 多层中断架构：串口接收中断、超声波外部边沿中断、定时器计时中断
- 传感器读取全部增加超时保护，杜绝卡死死机

## 快速编译 & 烧录教程
### 开发环境依赖
1. Keil MDK5（推荐 ARM Compiler 5）
2. STM32F4 标准外设库 V1.8.0
3. ST-Link / J-Link 调试下载器

### 使用步骤
1. 克隆本仓库到本地
2. 使用 Keil MDK5 打开 `project.uvprojx` 工程
3. 硬件适配修改（按需修改引脚宏）
   - `DHT11.h`：修改DHT11端口与引脚
   - `Motor.h`：麦克纳姆轮电机PWM、方向控制引脚
   - `HC_SR04.h`：超声波TRIG/ECHO引脚
4. 编译工程，无报错后通过下载器烧录至STM32F4开发板
5. 上电自检：LED闪烁3次，语音模块、BLE模块就绪，支持语音/安卓APP双模式控制小车运动

### 功能开关配置
1. **关闭自动避障**：将 `auto_avoid_flag = 0`，主循环停止超声波检测与避障动作
2. **开启温湿度串口上报**：取消main.c中DHT11周期读取代码注释，串口1定时输出温湿度
3. **关闭语音模块**：注释 `SU03T_Init()`，PC12输出低电平切断语音模块供电
4. **拓展麦克纳姆轮横移**：在Motor.c新增`Car_LeftShift()`/`Car_RightShift()`，调整四轮差速实现左右横移

## 关键模块原理说明
### 1. 麦克纳姆轮电机控制
TIM3 预分频83、重载999，PWM输出频率100Hz，速度区间 `0 ~ 1000`，四个电机独立调速，通过差速配合实现直行、原地旋转，可自行拓展斜向、横移运动。

### 2. DHT11 温湿度读取时序
主机输出19ms低电平起始信号，释放总线等待传感器应答；逐位读取40bit数据（湿度整数+湿度小数+温度整数+温度小数+校验字节），校验和匹配成功后解析0.1精度温湿度。

### 3. HC-SR04 超声波测距
TIM4 配置1us计数时钟，ECHO上升沿记录时间戳、下降沿停止计时；通过脉冲时长换算距离，内置定时器溢出补偿，测距稳定。障碍物小于25cm执行麦克纳姆轮专属后退绕行逻辑。

### 4. SU-03T 语音中断处理
UART4接收中断实时捕获语音模块返回单字节指令，无缓存延迟，直接调用电机/LED驱动函数，语音响应实时无卡顿。

### 5. BLE蓝牙通信对接
USART3与Ai-WB2-32S透传模块对接，严格匹配配套安卓APP的指令协议，APP下发字符指令控制小车运动，下位机定时推送温湿度、测距数据至手机端展示。
